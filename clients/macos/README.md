# Spotuify for macOS

A native SwiftUI player + menubar app that is a **client of the local `spotuify`
daemon** — the same daemon the CLI and TUI drive. It speaks the daemon's
length-delimited JSON IPC protocol over the Unix socket, renders daemon state, and
sends commands. It never owns playback state and never produces audio itself
(embedded librespot in the daemon does); optimistic UI is daemon-owned.

## Requirements

- macOS 26 (Tahoe) or later
- Xcode 26.3+ (Swift 6)
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) — `brew install xcodegen`
- A running `spotuify` daemon (`spotuify daemon start`)

## Project layout

This app uses **XcodeGen**: `project.yml` is the source of truth and generates
`Spotuify.xcodeproj`. The generated project IS committed (so the Xcode MCP and
`xcodebuild` work without anyone installing XcodeGen), but **re-run
`xcodegen generate` whenever you add or remove source files**, then commit the
updated project.

```
project.yml              # XcodeGen spec
Spotuify.xcodeproj/      # generated, committed
Support/Info.plist       # generated by XcodeGen from project.yml
Sources/
  SpotuifyKit/           # framework: IPC framing, wire models, stores, system bridges (testable)
  Spotuify/              # app: scenes + SwiftUI views
Tests/SpotuifyKitTests/  # unit tests (Swift Testing)
```

The app target is **non-sandboxed** so it can reach the daemon socket at
`~/Library/Application Support/<instance>/daemon.sock` (instance = `spotuify` for the
installed binary, `spotuify-dev` for cargo builds; `SPOTUIFY_SOCKET` overrides).

## Build & run

```bash
xcodegen generate
xcodebuild -project Spotuify.xcodeproj -scheme Spotuify -configuration Debug build
xcodebuild -project Spotuify.xcodeproj -scheme Spotuify -configuration Debug test
open "$(xcodebuild -project Spotuify.xcodeproj -scheme Spotuify -configuration Debug -showBuildSettings 2>/dev/null | awk -F' = ' '/ BUILT_PRODUCTS_DIR /{print $2}')/Spotuify.app"
```

## Local development loop

Use the helper script from the repo root or from `clients/macos`:

```bash
clients/macos/scripts/dev.sh check
clients/macos/scripts/dev.sh live-test
clients/macos/scripts/dev.sh run
```

`dev.sh run` launches the Debug app through `open` with:

```bash
SPOTUIFY_INSTANCE=spotuify-dev
SPOTUIFY_BIN=target/release/spotuify
```

That keeps local macOS edits pointed at the repo-built daemon instead of the installed Homebrew `spotuify` instance. The local workflow is CLI-only: edit files, then run:

```bash
clients/macos/scripts/dev.sh test
clients/macos/scripts/dev.sh run
```

The default test command disables live daemon tests. Use `clients/macos/scripts/dev.sh live-test` when you explicitly want to test the Swift IPC client against a running local `spotuify-dev` daemon. The live-test scheme forces `SPOTUIFY_INSTANCE=spotuify-dev`, so it will not accidentally use the installed/Homebrew daemon.

If you add or remove Swift source files, run `clients/macos/scripts/dev.sh generate` and commit the regenerated `Spotuify.xcodeproj`.

## Verifying against the daemon

This app is a client, so the integration test is: drive the daemon from a terminal and
watch the app react.

```bash
spotuify daemon status         # confirm the daemon is up
spotuify play "..."            # app should show the track + start ticking
spotuify pause                 # app transport reflects paused
spotuify next                  # app advances
```
